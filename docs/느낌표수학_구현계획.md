### 접근성 규칙 (필수)
- **노랑 글자를 흰/밝은 배경 위에 쓰지 않는다** (대비 미달). 밝은 배경에서 노랑은 채움·테두리·마크·칩 배경으로만.
- 밝은 배경 위 강조 텍스트(전화번호, eyebrow, `.hl` 등)는 **검정**으로. 필요 시 노랑 밑줄/하이라이터(배경 노랑+글자 검정) 사용.
- 검정 배경(목표 밴드·콜밴드·contact) 위 하이라이트는 **노랑 글자** — 여기서만 노랑 텍스트 허용.
- 시안에서 파랑 텍스트였던 곳별 처리:
  - `.hd-tel`(헤더 전화) → 검정 텍스트 (또는 노랑 배경 + 검정 글자 pill)
  - `.eyebrow` → 검정 텍스트 + 앞 `!` 마크만 노랑 배경 처리
  - `.goal .hl`, `.contact .hl` (다크 배경) → 노랑 텍스트
  - `.part-card h4 .pn`(번호 원) → 노랑 배경 + 검정 숫자
  - `.tag`(풀이설계력 훈련 칩) → 노랑 배경 + 검정 글자
  - `.step::before`(입학 스텝 번호) → 연노랑 배경 + 검정 테두리·검정 숫자
  - 워터마크 `!`(히어로·목표밴드) → 노랑, 기존 opacity 유지
- 다크 밴드의 보조 텍스트 색(#AFC3D4, #C4D4E2, #9FB6C9 등 블루그레이 계열) → 웜그레이/옅은 노랑기 계열(#D9D5C7, #C9C5B8 등)로 교체

## 6. 메인 페이지 구현 상세

시안 HTML의 마크업을 섹션별 컴포넌트로 분해해 그대로 이식한다. 카피는 시안(=기획서 v1.5)에서 복사하며 **수정 금지**.

### 6-1. 시맨틱 마크업 (SEO 필수)
- `<h1>` 페이지당 정확히 1개 — 메인은 "풀이를 설계하는 아이로 키웁니다" (슬로건의 '아이' 유지가 최종본)
- 섹션 제목 `<h2>`, 블록·파트 제목 `<h3>`/`<h4>` — 계층 건너뛰기 금지
- `<header>` `<main>` `<section>` `<nav>` `<footer>` 사용
- 모든 본문은 실제 텍스트 (이미지 속 글자 금지)
- 전화번호는 전부 `<a href="tel:0319121538">` — 노출 4곳: 헤더 상시 / 프로그램 뒤 콜밴드 / 입학 1단계 / 오시는 길
- 학교명(한수중·오마중·발산중·대화중·장성중·신일중)은 텍스트 칩으로 유지
- 이미지 alt에 지역·학원 맥락 포함 (예: "주엽동 느낌표수학학원 로고")

### 6-2. Header (client)
- sticky, blur 배경, 로고 + 전화(tel:) + 햄버거 버튼
- 햄버거: `useState` 토글, nav 링크 클릭 시 닫힘, `aria-expanded`/`aria-label`
- 로고: 노랑·검정 심볼 파일 확보 전에는 텍스트 로고 `느낌표수학<span class="bang">!</span>` (`.bang`은 노랑)

### 6-3. Reveal (client)
- 시안의 `.reveal` 스크롤 리빌을 래퍼 컴포넌트화: IntersectionObserver(threshold .12), 진입 시 `.on` 부여 후 unobserve
- `prefers-reduced-motion: reduce` 시 애니메이션 제거 (CSS에 이미 있음 — 유지)
